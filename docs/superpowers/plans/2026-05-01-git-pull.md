# Git Pull Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add git pull support to the existing GitStatusMenu — a left-side badge showing commits-behind count, a pull button in the dropdown, background polling via git fetch, and a settings field to configure the poll interval.

**Architecture:** Client-side setInterval fires `POST /api/projects/:id/git/fetch` at the configured interval, then reads the updated `behind` count from the existing `GET /api/projects/:id/git/status`. A new `POST /api/projects/:id/git/pull` endpoint handles the actual pull. The GitStatusMenu component owns the polling timer, starts/resets it on project switch, and displays the badge and pull controls.

**Tech Stack:** Express (server endpoints), TypeScript, React 19 + Zustand (state/UI)

app.post("/api/projects/:id/git/fetch", async (request, response) => {
  const project = projectById(request.params.id);
  if (!project) {
    response.status(404).json({ error: "Project not found." });
    return;
  }
  try {
    await gitCommand(project, ["fetch"], 120000, {
      env: { GIT_TERMINAL_PROMPT: "0" },
      timeoutMessage: "Git fetch timed out."
    });
    response.json({ ok: true });
  } catch (error) {
    response.status(500).json({ error: error instanceof Error ? error.message : String(error) });
  }
});

    if (!projectId) return;
    setLoading(true);
    try {

    } catch {
      // silently ignore fetch errors — use last known status
    }
    try {
      setStatus(await api.gitStatus(projectId));
    } catch (error) {
      addError(error instanceof Error ? error.message : String(error));
    } finally {
      setLoading(false);
    }
